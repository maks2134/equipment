# Инструкция по запуску проекта

## Система учета оборудования

Приложение для учета оборудования с поддержкой различных типов техники, пользователей и ролей.

---

## Требования

- **Windows 10** версии 1809 (17763) или выше
- **.NET 8.0 SDK** или выше
- **Visual Studio 2022** (рекомендуется) или Visual Studio Code
- **Windows App SDK 1.8** или выше

---

## Установка зависимостей

### 1. Установка .NET SDK

1. Скачайте и установите .NET 8.0 SDK с официального сайта: https://dotnet.microsoft.com/download
2. Проверьте установку, выполнив в командной строке:
   ```bash
   dotnet --version
   ```
   Должна отобразиться версия 8.0.x или выше.

### 2. Установка Visual Studio (рекомендуется)

1. Скачайте Visual Studio 2022 Community (бесплатная версия) с официального сайта: https://visualstudio.microsoft.com/
2. При установке выберите рабочую нагрузку **"Разработка классических приложений .NET"** и **"Разработка приложений для UWP"**
3. Убедитесь, что установлен компонент **Windows 10 SDK** (версия 10.0.19041.0 или выше)

---

## Открытие проекта в Visual Studio

### Способ 1: Через Visual Studio

1. Запустите **Visual Studio 2022**
2. Выберите **"Открыть проект или решение"** (Open a project or solution)
3. Перейдите в папку с проектом и выберите файл `equipment_accounting_system.csproj`
4. Дождитесь восстановления пакетов NuGet (автоматически)

### Способ 2: Через командную строку

1. Откройте командную строку или PowerShell
2. Перейдите в папку проекта:
   ```bash
   cd путь\к\проекту\equipment_accounting_system
   ```
3. Восстановите зависимости:
   ```bash
   dotnet restore
   ```
4. Откройте проект в Visual Studio:
   ```bash
   start equipment_accounting_system.csproj
   ```

---

## Запуск проекта

### Способ 1: Запуск из Visual Studio

1. Убедитесь, что в верхней панели выбран **Debug** или **Release** режим
2. Выберите платформу: **x64** (рекомендуется) или **x86**
3. Нажмите клавишу **F5** или кнопку **"Запустить"** (Start)
4. Приложение откроется в новом окне

### Способ 2: Запуск из командной строки

1. Откройте командную строку или PowerShell в папке проекта
2. Выполните команду:
   ```bash
   dotnet run
   ```
   или для конкретной платформы:
   ```bash
   dotnet run -c Release -r win-x64
   ```

---

## Первый запуск

При первом запуске приложения:

1. **Окно авторизации** - введите учетные данные:
   - **Администратор**: 
     - Логин: `admin`
     - Пароль: `admin`
   - **Пользователь**:
     - Логин: `user`
     - Пароль: `user`

2. После входа вы увидите главное окно приложения с пустым списком оборудования

3. **Создание данных**:
   - Нажмите кнопку **"Добавить"** для создания нового оборудования
   - Выберите категорию и тип оборудования
   - Заполните необходимые поля
   - Сохраните

4. **Данные сохраняются** автоматически в папку `data` в директории проекта:
   - `data/equipment.json` - список оборудования
   - `data/users.json` - список пользователей

---

## Регистрация новых пользователей

1. В окне авторизации нажмите кнопку **"Регистрация"**
2. Заполните форму:
   - Имя пользователя (минимум 3 символа)
   - Пароль (минимум 3 символа)
   - Подтверждение пароля
   - Выберите роль: **Пользователь** или **Администратор**
3. Нажмите **"Зарегистрироваться"**
4. Теперь можно войти с новыми учетными данными

---

## Управление пользователями (только для администраторов)

1. Войдите как администратор
2. В главном окне нажмите кнопку **"Пользователи"** (видна только администраторам)
3. В открывшемся окне можно:
   - Просмотреть список всех пользователей
   - Добавить нового пользователя
   - Удалить пользователя (нельзя удалить последнего администратора)

---

## Работа с оборудованием

### Добавление оборудования

1. Нажмите кнопку **"Добавить"**
2. Выберите **категорию оборудования** (Компьютерное, Офисное, Мобильное)
3. Выберите **конкретный тип** из списка подтипов
4. Заполните общие поля:
   - Название
   - Производитель
   - Цена
   - Дата покупки
   - Местоположение
   - Статус
5. Заполните специфичные поля для выбранного типа
6. Нажмите **"Сохранить"**

### Редактирование оборудования

1. Выберите оборудование в списке
2. Нажмите кнопку **"Редактировать"**
3. Внесите изменения
4. Нажмите **"Сохранить"**

### Удаление оборудования

1. Выберите оборудование в списке
2. Нажмите кнопку **"Удалить"**
3. Подтвердите удаление

### Поиск и фильтрация

- Используйте поле **"Поиск по названию"** для быстрого поиска
- Используйте **"Фильтр по статусу"** для отображения оборудования с определенным статусом
- Используйте **"Сортировка"** для упорядочивания списка

## Типы оборудования

### Компьютерное оборудование
- **Компьютерное оборудование** (базовый тип)
- **Настольный компьютер** - с указанием форм-фактора, блока питания
- **Ноутбук** - с указанием размера экрана, времени работы от батареи
- **Рабочая станция** - с указанием видеокарты, видеопамяти
- **Серверное оборудование** - с указанием высоты в стойке, резервного питания
- **Сетевое оборудование** - с указанием количества портов, типа подключения

### Офисное оборудование
- **Офисное оборудование** (базовый тип)
- **Принтер** - с указанием типа принтера, размера бумаги
- **Сканер** - с указанием разрешения, типа сканера
- **Копир** - с указанием скорости копирования, емкости лотка
- **Факс** - с указанием скорости передачи, памяти

### Мобильное оборудование
- **Мобильное оборудование** (базовый тип)
- **Планшет** - с указанием поддержки клавиатуры, стилуса
- **Смартфон** - с указанием камеры, встроенной памяти, NFC

## Решение проблем

### Ошибка при запуске: "Не удалось найти SDK"

1. Убедитесь, что установлен **Windows 10 SDK** версии 10.0.19041.0 или выше
2. В Visual Studio: **Инструменты** → **Получить средства и функции** → установите Windows 10 SDK

### Ошибка: "Packaged .NET applications cannot be ProcessorArchitecture neutral"

1. В Visual Studio выберите конкретную платформу: **x64** или **x86** (не AnyCPU)
2. Или укажите RuntimeIdentifier в командной строке:
   ```bash
   dotnet build -r win-x64
   ```

### Приложение не запускается

1. Проверьте, что установлен **.NET 8.0 Runtime**
2. Убедитесь, что Windows 10 обновлена до версии 1809 или выше
3. Проверьте логи ошибок в окне вывода Visual Studio

### Данные не сохраняются

1. Убедитесь, что у приложения есть права на запись в папку проекта
2. Проверьте наличие папки `data` в директории проекта
3. Проверьте файлы `data/equipment.json` и `data/users.json`

## Сборка исполняемого файла (.exe)

### Способ 1: Через Visual Studio

1. Откройте проект в Visual Studio
2. Выберите конфигурацию **Release**
3. Выберите платформу **x64** (или x86)
4. В меню выберите: **Сборка** → **Опубликовать** (Build → Publish)
5. Выберите папку для публикации
6. Нажмите **"Опубликовать"** (Publish)
7. Исполняемый файл будет в папке `bin\Release\net8.0-windows10.0.19041.0\win-x64\publish\`

### Способ 2: Через командную строку

1. Откройте командную строку или PowerShell в папке проекта
2. Выполните команду:
   ```bash
   dotnet publish -c Release -r win-x64 --self-contained false
   ```
   Для создания автономного приложения (со всеми зависимостями):
   ```bash
   dotnet publish -c Release -r win-x64 --self-contained true
   ```
3. Исполняемый файл будет в папке:
   ```
   bin\Release\net8.0-windows10.0.19041.0\win-x64\publish\
   ```

### Способ 3: Создание установщика (MSIX)

1. В Visual Studio выберите: **Проект** → **Упаковать** (Project → Package)
2. Следуйте инструкциям мастера
3. Будет создан файл `.msix` для установки приложения
